# 动态天气仪表板 🌤️

实时展示中国各大城市或区域的天气信息，支持自定义城市管理、自动解析地理位置及自定义排序。

## ✨ 功能特点

- 🌍 **动态城市管理**：支持自由新增和删除城市，不再局限于固定城市。
- 🔍 **智能地址解析**：输入中文城市名或区域名（如“北京”、“周浦”），系统自动解析地理坐标。
- 🔢 **自定义排序**：支持手动调整城市卡片顺序，优先查看关心的内容。
- 🔄 **自动更新**：每 5 分钟自动刷新天气数据，保持信息实时。
- 💾 **本地持久化**：城市列表和排序状态自动保存至浏览器 local storage，刷新后依然有效。
- 🎨 **现代设计**：玻璃态效果、渐变背景、流畅动画。
- 📱 **响应式布局**：完美适配桌面、平板和手机。

## 🛠️ 技术栈

- **前端**：HTML5, CSS3, Vanilla JavaScript
- **API**：OpenWeatherMap API (Weather & Geocoding)
- **容器**：Netlify (支持自动部署)

## 📦 项目结构

```
weather-dashboard/
├── index.html          # 主页面（动态渲染）
├── style.css          # 核心样式
├── script.js          # 核心逻辑（数据管理、API交互、排序）

```

## 🚀 快速开始

1. 直接在浏览器中打开 `index.html`。
2. 点击顶部的 **“添加城市”** 按钮。
3. 输入你想关注的中国城市或地区名，点击搜索。
4. 系统会自动解析地址并添加到你的仪表板。
5. 使用卡片上的 **↑** **↓** 按钮进行排序，或点击 **✕** 删除城市。

## 🔑 API 配置

本项目使用的是 OpenWeatherMap API。在 `script.js` 中：
- `API_KEY`: 已配置内置密钥，也支持自定义。
- `API_BASE_URL`: 实时天气接口。
- `Geocoding API`: 用于将中文地址转换为经纬度坐标。

## 🌐 部署

每次将代码推送到 GitHub `main` 分支，Netlify 将会自动触发构建并更新线上版本。
